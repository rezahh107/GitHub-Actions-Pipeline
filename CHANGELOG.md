# Changelog

## 0.2.0

- Add explicit `minimal-safe-ci` and `deep-repository-upgrade` operating modes.
- Add a manifest-aware, component-aware repository model.
- Parse bounded workflow triggers, jobs, permissions, actions, and executed command signals.
- Add historical analysis for reverts, co-change pairs, source changes without adjacent tests, repeated fixes, and workflow/configuration churn.
- Add independent recommendation channels for observed failures, structural invariants, and baseline capabilities.
- Add cold-start behavior, composable capability profiles, operational capability states, and bounded ordinal ranking.
- Add staged Phase 1/Phase 2 upgrade reports with deterministic hashing and Persian summaries.
- Add schemas, examples, negative tests, mode-separation tests, and a durable implementation-status contract.
- Preserve the existing conservative CI workflow and evidence-honesty rules.

## 0.1.1

- Prevent shallow Git history from being reported as complete.
- Add explicit bounded, partial, unavailable, unknown, and not-applicable evidence states.
- Separate tested SHA, source head SHA, base SHA, and tested ref kind.
- Add canonical evidence SHA-256 and reproducible JSON serialization.
- Close evidence-item schemas and align producer output with fixtures.
- Add deferred gates, structured risk assessment, and execution contracts to the CI Gate Map.
- Remove the duplicate hand-maintained YAML example.
- Add deterministic, shallow-clone, CLI, schema-drift, and workflow-contract tests.
- Bound validation runtime and artifact retention while keeping product Markdown in CI scope.
